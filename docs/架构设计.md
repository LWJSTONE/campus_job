# 校园兼职网站 - 系统架构设计文档

## 1. 系统概述

### 1.1 项目背景
校园兼职网站旨在为高校学生和企业商家提供一个安全、高效、透明的兼职信息对接平台，解决传统兼职市场中信息不对称、流程复杂、缺乏监管等问题。

### 1.2 技术选型理由

| 技术选型 | 选择原因 |
|---------|---------|
| **Spring Cloud** | 成熟的微服务框架，生态完善，社区活跃 |
| **Spring Boot 3.2** | 快速开发，自动配置，内置Tomcat |
| **Vue 3 + TypeScript** | 现代化前端框架，类型安全，易于维护 |
| **MyBatis Plus** | 简化CRUD操作，支持代码生成 |
| **MySQL 8.0** | 开源关系型数据库，性能优秀 |
| **Redis 7.x** | 高性能缓存，支持多种数据结构 |
| **RabbitMQ** | 可靠的消息队列，支持异步处理 |
| **Docker** | 容器化部署，环境一致性 |

## 2. 微服务架构设计

### 2.1 服务划分

| 服务名称 | 端口 | 职责描述 |
|---------|------|---------|
| **gateway** | 10000 | API网关，统一入口，路由转发，权限控制 |
| **eureka-server** | 8761 | 服务注册与发现中心 |
| **config-server** | 8888 | 统一配置中心 |
| **auth-service** | 8001 | 用户认证，JWT令牌生成与验证 |
| **user-service** | 8002 | 用户管理（学生、企业、管理员） |
| **job-service** | 8003 | 兼职岗位发布、搜索、推荐 |
| **application-service** | 8004 | 兼职申请管理、审核流程 |
| **message-service** | 8005 | 消息通知、即时通讯 |

### 2.2 服务依赖关系

```
用户请求
  ↓
Gateway (网关)
  ↓
Auth Service (认证) → Eureka (服务发现)
  ↓                    ↓
Config (配置) → 各微服务 (user/job/application/message)
```

## 3. 核心模块设计

### 3.1 网关服务 (gateway)

**功能职责：**
- 统一API入口
- 路由转发（根据URL路径转发到不同微服务）
- 身份认证（JWT验证）
- 权限控制
- 限流熔断
- 跨域处理

**核心配置：**
```yaml
spring:
  cloud:
    gateway:
      routes:
        - id: auth-service
          uri: lb://auth-service
          predicates:
            - Path=/api/auth/**
          filters:
            - StripPrefix=2
        
        - id: user-service
          uri: lb://user-service
          predicates:
            - Path=/api/user/**
          filters:
            - StripPrefix=2
```

### 3.2 认证服务 (auth-service)

**功能职责：**
- 用户登录验证
- JWT令牌生成
- 令牌刷新
- 第三方登录集成
- 验证码生成与验证

**核心流程：**
```
用户提交登录信息
  ↓
验证用户名密码
  ↓
生成JWT Token (包含用户ID、角色、权限)
  ↓
返回Token给客户端
  ↓
客户端在请求头携带Token
  ↓
网关验证Token并放行/拦截
```

**JWT Token结构：**
```json
{
  "header": {
    "alg": "HS256",
    "typ": "JWT"
  },
  "payload": {
    "sub": "user_id",
    "userId": 123,
    "username": "student01",
    "roles": ["STUDENT"],
    "iat": 1234567890,
    "exp": 1234571490
  },
  "signature": "签名"
}
```

### 3.3 用户服务 (user-service)

**功能职责：**
- 用户注册（学生、企业）
- 用户信息管理
- 个人信息修改
- 密码重置
- 用户资料完善
- 企业资质审核

**数据模型：**
```java

// 学生信息扩展

    @Id

// 企业信息扩展

    @Id
```

### 3.4 兼职服务 (job-service)

**功能职责：**
- 兼职岗位发布
- 岗位搜索与筛选
- 岗位分类管理
- 智能推荐（协同过滤）
- 岗位浏览记录
- 收藏功能

**数据模型：**
```java
@Entity
@Table(name = "job")
public class Job {
    @Id
    private Long id;
    private Long companyId;  // 发布企业ID
    private String title;
    private String description;
    private String category;  // 分类：家教/促销/翻译/设计等
    private BigDecimal salary;  // 薪资
    private String salaryType;  // 时薪/日薪/月薪
    private String workTime;
    private String workPlace;
    private Integer recruitCount;  // 招聘人数
    private Integer currentCount; // 已招人数
    private Integer status;  // 0-待审核 1-已发布 2-已下架
    private LocalDateTime publishTime;
    private LocalDateTime deadline;
    
    // 协同推荐相关
    @Transient
    private Double similarityScore;  // 相似度分数
}

@Entity
@Table(name = "job_category")
public class JobCategory {
    @Id
    private Long id;
    private String name;
    private String icon;
    private Integer sortOrder;
}

@Entity
@Table(name = "job_favorite")
public class JobFavorite {
    @Id
    private Long id;
    private Long userId;
    private Long jobId;
    private LocalDateTime createTime;
}
```

**协同过滤推荐算法：**
```
基于用户的协同过滤（User-based CF）：
1. 计算用户之间的相似度（余弦相似度）
2. 找到与目标用户最相似的K个用户
3. 推荐这些相似用户喜欢但目标用户未浏览的兼职

基于物品的协同过滤（Item-based CF）：
1. 计算兼职之间的相似度（基于用户的浏览/申请行为）
2. 找到与目标兼职最相似的兼职
3. 推荐相似的兼职

本项目采用混合推荐策略：
- 新用户：热门兼职推荐
- 老用户：协同过滤推荐
```

### 3.5 申请服务 (application-service)

**功能职责：**
- 兼职申请提交
- 申请审核（企业审核）
- 申请状态管理
- 申请记录查询
- 简历投递
- 面试安排

**申请状态流转：**
```
PENDING (待审核)
  ↓
APPROVED (已通过) → HIRED (已录用) → COMPLETED (已完成)
  ↓
REJECTED (已拒绝)
```

**数据模型：**
```java
@Entity
@Table(name = "job_application")
public class JobApplication {
    @Id
    private Long id;
    private Long jobId;
    private Long studentId;
    private String resume;  // 简历URL
    private String selfIntroduction;  // 自我介绍
    private Integer status;  // 0-待审核 1-已通过 2-已拒绝 3-已录用 4-已完成
    private String remark;  // 审核备注
    private LocalDateTime applyTime;
    private LocalDateTime auditTime;
}

@Entity
@Table(name = "job_record")
public class JobRecord {
    @Id
    private Long id;
    private Long jobId;
    private Long studentId;
    private LocalDateTime startTime;
    private LocalDateTime endTime;
    private BigDecimal actualSalary;  // 实际薪资
    private Integer status;  // 0-进行中 1-已完成
    private String comment;  // 工作评价
}
```

## 5. 安全设计

### 5.1 认证流程

```
1. 用户登录 → Auth Service验证 → 生成JWT
2. 客户端请求携带JWT → Gateway验证JWT → 放行/拦截
3. JWT包含用户ID、角色、权限信息
4. Service从JWT中获取用户信息进行权限校验
```

### 5.2 权限控制

**角色定义：**
- **ADMIN**: 系统管理员，全部权限
- **COMPANY**: 企业用户，发布兼职、审核申请
- **STUDENT**: 学生用户，浏览、申请兼职

**权限注解使用：**
```java
@PreAuthorize("hasRole('ADMIN')")
public void deleteUser(Long userId) { }

@PreAuthorize("hasRole('COMPANY')")
public void publishJob(Job job) { }

@PreAuthorize("hasRole('STUDENT')")
public void applyJob(Long jobId) { }
```

### 5.3 数据安全

- 密码使用BCrypt加密存储
- 敏感信息传输使用HTTPS
- SQL注入防护（MyBatis参数化查询）
- XSS攻击防护（前端输入过滤）
- CSRF防护（Spring Security）

## 10. 技术难点与解决方案

### 10.1 分布式事务

**问题：** 跨服务的事务一致性

**解决方案：** 最终一致性 + 消息队列
```
用户申请兼职 → 发送消息到MQ → MQ保证消息投递 → 订阅者处理消息
```

### 10.2 服务间调用

**问题：** 服务雪崩效应

**解决方案：** 
- 熔断器（Hystrix/Sentinel）
- 超时控制
- 限流保护

### 10.3 推荐算法

**问题：** 冷启动问题（新用户/新兼职）

**解决方案：**
- 新用户：热门兼职推荐
- 新兼职：基于内容推荐
- 混合策略：多种算法组合

## 11. 总结

本系统采用Spring Cloud微服务架构，实现了服务解耦、独立部署、易于扩展。通过合理的模块划分和架构设计，保证了系统的可维护性、可扩展性和高性能。
