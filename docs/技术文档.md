### 微服务拆分原则

基于领域驱动设计（DDD），系统按业务领域拆分为以下微服务：

1. **Auth Service（认证服务）**
   - 职责：用户认证、Token 管理、权限控制
   - 领域：认证授权

2. **User Service（用户服务）**
   - 职责：用户信息管理、学生/企业信息维护
   - 领域：用户管理

3. **Job Service（兼职服务）**
   - 职责：兼职信息发布、搜索、推荐
   - 领域：兼职管理

4. **Application Service（申请服务）**
   - 职责：兼职申请、审核、流程管理
   - 领域：申请管理

5. **Message Service（消息服务）**
   - 职责：站内信、通知、消息推送
   - 领域：消息管理

6. **Other Services（其他服务）**
   - 职责：评价、统计、支付等（预留扩展）
   - 领域：未来扩展

## 数据库设计

### 数据库规范

1. **命名规范**
   - 表名：小写字母，单词间用下划线分隔（如：`user_info`）
   - 字段名：小写字母，单词间用下划线分隔
   - 索引名：`idx_` + 表名 + 字段名（如：`idx_user_mobile`）
   - 唯一索引：`uk_` + 表名 + 字段名（如：`uk_user_username`）

2. **字段规范**
   - 主键：统一使用 `id`（BIGINT 自增）
   - 创建时间：`created_at`（DATETIME）
   - 更新时间：`updated_at`（DATETIME，自动更新）
   - 状态字段：使用 TINYINT，0 表示禁用/失败，1 表示启用/成功

3. **字符集规范**
   - 字符集：`utf8mb4`
   - 排序规则：`utf8mb4_unicode_ci`

### 核心表结构

#### 1. 用户表（users）

```sql
CREATE TABLE `users` (
  `id` bigint NOT NULL AUTO_INCREMENT COMMENT '用户ID',
  `username` varchar(50) NOT NULL COMMENT '用户名',
  `password` varchar(100) NOT NULL COMMENT '密码(BCrypt加密)',
  `user_type` tinyint NOT NULL DEFAULT '0' COMMENT '用户类型(0-学生,1-企业,2-管理员)',
  `status` tinyint NOT NULL DEFAULT '1' COMMENT '状态(0-禁用,1-正常)',
  `created_at` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_username` (`username`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';
```

## 开发规范

### 代码规范

#### 1. Java 代码规范

遵循阿里巴巴 Java 开发手册：

- 类名使用大驼峰命名（如：`UserService`）
- 方法名使用小驼峰命名（如：`getUserInfo`）
- 常量使用全大写下划线分隔（如：`MAX_SIZE`）
- 包名全小写，使用点号分隔

#### 2. Vue 代码规范

- 组件名使用大驼峰命名（如：`UserProfile.vue`）
- 变量名使用小驼峰命名
- 常量使用全大写下划线分隔
- 使用 TypeScript 进行类型检查

#### 3. 数据库规范

- 表名使用小写字母和下划线
- 字段名使用小写字母和下划线
- 必须有主键、创建时间、更新时间
- 必须有注释说明

### Git 提交规范

采用 Angular 提交规范：

```
<type>(<scope>): <subject>

<body>

<footer>
```

**type 类型：**
- `feat`: 新功能
- `fix`: 修复 bug
- `docs`: 文档更新
- `style`: 代码格式（不影响功能）
- `refactor`: 重构
- `test`: 测试
- `chore`: 构建/工具链更新

**示例：**
```
feat(auth): 添加 JWT 登录功能

- 实现 JWT 令牌生成与验证
- 添加登录接口
- 更新文档

Closes #123
```

### 接口设计规范

1. RESTful 风格
2. 统一响应格式
3. 合理使用 HTTP 状态码
4. 接口版本控制（如：`/api/v1/xxx`）
5. 接口文档维护

## 性能优化

### 缓存策略

1. **Redis 缓存**
   - 热点数据缓存（如：兼职列表、用户信息）
   - 缓存过期策略：30分钟
   - 缓存更新策略：Cache-Aside

2. **本地缓存**
   - 使用 Caffeine 作为本地缓存
   - 缓存配置信息、字典数据

### 数据库优化

1. **索引优化**
   - 为常用查询字段添加索引
   - 避免全表扫描

2. **读写分离**
   - 主库写，从库读
   - 使用 ShardingSphere 实现读写分离

3. **分库分表**
   - 按业务分库
   - 大表分表（如：申请记录表按时间分表）

### 异步处理

使用 RabbitMQ 实现异步处理：

- 消息通知异步发送
- 数据统计异步计算
- 日志异步写入

## 安全设计

### 认证与授权

1. **JWT 认证**
   - 无状态令牌
   - 令牌有效期：24小时
   - 刷新令牌有效期：7天

2. **权限控制**
   - 基于角色的访问控制（RBAC）
   - 接口级别的权限控制

### 数据安全

1. **密码加密**
   - 使用 BCrypt 加密存储密码
   - 加密强度：10轮

2. **敏感信息加密**
   - 手机号、身份证号等脱敏显示
   - 使用 AES 加密存储

3. **HTTPS**
   - 生产环境强制使用 HTTPS
   - 证书自动续期

### 防 SQL 注入

- 使用 MyBatis Plus 参数化查询
- 禁止 SQL 拼接

### 防 XSS 攻击

- 前端输入验证
- 输出转义
- 使用 CSP (Content Security Policy)

### 防 CSRF 攻击

- 使用 Token 验证
- SameSite Cookie
