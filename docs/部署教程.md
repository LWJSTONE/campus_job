## 前置要求

### 硬件要求

- **操作系统**：Windows 10/11、macOS、Linux (Ubuntu 20.04+)
- **内存**：建议 8GB 及以上
- **磁盘空间**：至少 20GB 可用空间

### 软件要求

在开始部署之前，请确保已安装以下软件：

1. **Git**（用于克隆代码）
   - Windows：下载 [Git for Windows](https://git-scm.com/download/win)
   - macOS：`brew install git`
   - Linux：`sudo apt-get install git`

2. **Docker Desktop**（核心容器化平台）
   - Windows/macOS：下载 [Docker Desktop](https://www.docker.com/products/docker-desktop)
   - Linux：安装 Docker Engine 和 Docker Compose

3. **Java JDK 17**（用于编译后端代码，可选）
   - 下载 [Oracle JDK 17](https://www.oracle.com/java/technologies/downloads/)
   - 或使用 OpenJDK 17

4. **Node.js 18**（用于编译前端代码，可选）
   - 下载 [Node.js](https://nodejs.org/)

5. **Maven 3.8+**（用于构建后端，可选）
   - 下载 [Apache Maven](https://maven.apache.org/download.cgi)

> **注意**：如果项目已提供预编译的 JAR 包，可以跳过 Java 和 Maven 的安装。

## 快速开始

### 1. 克隆项目代码

```bash
# 使用 HTTPS 克隆（推荐）
git clone https://github.com/LWJSTONE/campus_job.git

# 或使用 SSH 克隆（需要配置 SSH 密钥）
git clone git@github.com:LWJSTONE/campus_job.git

# 进入项目目录
cd campus_job
```

### 2. 启动 Docker Desktop

确保 Docker Desktop 已启动并正常运行：

- **Windows/macOS**：在应用程序菜单中启动 Docker Desktop，等待图标变为运行状态
- **Linux**：执行 `sudo systemctl start docker`

验证 Docker 是否正常运行：

```bash
docker version
docker-compose version
```

### 3. 一键部署

```bash
# 启动所有服务
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f
```

### 4. 访问系统

等待 2-3 分钟，待所有服务启动完成后，访问以下地址：

- **前端系统**：http://localhost
- **Eureka 控制台**：http://localhost:8761
- **API 网关**：http://localhost:10000
- **RabbitMQ 管理界面**：http://localhost:15672（账号：admin/admin123）

## 详细部署步骤

### 步骤 1：环境准备

#### 1.1 安装 Docker Desktop

**Windows：**
1. 下载 Docker Desktop Installer
2. 双击安装包，按照提示完成安装
3. 重启计算机
4. 启动 Docker Desktop，确保状态为 "Docker Desktop is running"

**macOS：**
1. 下载 Docker.dmg 文件
2. 将 Docker 拖拽到 Applications 文件夹
3. 启动 Docker Desktop
4. 在菜单栏中，确保 Docker 图标显示运行状态

**Linux (Ubuntu)：**
```bash
# 更新包索引
sudo apt-get update

# 安装依赖
sudo apt-get install -y apt-transport-https ca-certificates curl software-properties-common

# 添加 Docker 官方 GPG 密钥
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -

# 添加 Docker 仓库
sudo add-apt-repository "deb [arch=amd64] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable"

# 安装 Docker
sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# 启动 Docker
sudo systemctl start docker
sudo systemctl enable docker

# 将当前用户添加到 docker 组（避免每次使用 sudo）
sudo usermod -aG docker $USER
newgrp docker
```

#### 1.2 配置 Docker 镜像加速器（可选，中国大陆用户推荐）

打开 Docker Desktop 设置：

1. 点击 Docker 图标，选择 **Settings**
2. 进入 **Docker Engine**
3. 在 JSON 配置中添加以下内容：

```json
{
  "registry-mirrors": [
    "https://docker.mirrors.ustc.edu.cn",
    "https://hub-mirror.c.163.com"
  ]
}
```

4. 点击 **Apply & Restart**

## 常见问题

### 问题 1：Docker 启动失败

**错误信息**：`Cannot connect to the Docker daemon`

**解决方案**：
```bash
# 检查 Docker 是否运行
sudo systemctl status docker

# 启动 Docker
sudo systemctl start docker

# macOS/Windows：检查 Docker Desktop 是否启动
```

### 问题 2：端口被占用

**错误信息**：`Bind for 0.0.0.0:3306 failed: port is already allocated`

**解决方案**：
```bash
# 查看端口占用情况
netstat -ano | findstr "3306"  # Windows
lsof -i :3306                  # macOS/Linux

# 方法 1：停止占用端口的程序
# 方法 2：修改 docker-compose.yml 中的端口映射
# 例如，将 MySQL 端口从 3306 改为 3307
ports:
  - "3307:3306"
```

### 问题 3：容器启动失败

**错误信息**：容器反复重启

**解决方案**：
```bash
# 查看容器日志
docker-compose logs <service-name>

# 例如，查看 MySQL 日志
docker-compose logs mysql

# 根据日志信息排查问题
```

### 问题 4：服务间无法通信

**解决方案**：
- 检查所有服务是否在同一个网络中：`docker network ls`
- 检查服务名称是否正确（在 docker-compose.yml 中定义的服务名）
- 确认依赖关系（depends_on）配置正确

### 问题 5：内存不足

**错误信息**：`no space left on device` 或容器被杀死

**解决方案**：
```bash
# 清理未使用的镜像和容器
docker system prune -a

# 增加 Docker Desktop 的内存限制
# macOS/Windows：Docker Desktop -> Settings -> Resources -> Memory
```

## 开发模式部署

如果需要进行开发调试，可以分别启动各个服务：

### 启动基础服务

```bash
# 只启动数据库、缓存、消息队列
docker-compose up -d mysql redis rabbitmq
```

### 本地运行后端

```bash
# 进入后端目录
cd backend

# 启动 Eureka Server
cd eureka-server
mvn spring-boot:run

# 新开终端，启动 Gateway
cd ../gateway
mvn spring-boot:run

# 新开终端，启动其他服务...
```

### 本地运行前端

```bash
# 进入前端目录
cd frontend

# 安装依赖
npm install

# 启动开发服务器
npm run dev

# 访问：http://localhost:5173
```

## 停止和清理

### 停止服务

```bash
# 停止所有服务
docker-compose stop

# 停止并删除容器
docker-compose down

# 停止并删除容器、网络、卷
docker-compose down -v
```

### 清理资源

```bash
# 清理未使用的镜像
docker image prune -a

# 清理未使用的容器
docker container prune

# 清理未使用的卷
docker volume prune

# 清理所有未使用的资源
docker system prune -a --volumes
```
